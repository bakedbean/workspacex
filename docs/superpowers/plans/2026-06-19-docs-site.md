# wsx Documentation Site (mdBook) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single 1351-line `README.md` with a navigable, searchable mdBook documentation site auto-published to GitHub Pages, and slim the README to a landing page.

**Architecture:** An mdBook project at `docs/book/` holds the docs as ~30 focused markdown pages (one per current README H3), with `SUMMARY.md` as the nav. Content migrates near-verbatim from the README; the README is then slimmed to a landing page that links to the site. A GitHub Actions workflow builds the book and deploys it to GitHub Pages on every push to `main`. The book is the single source of truth for detailed docs.

**Tech Stack:** mdBook (Rust, built-in search), GitHub Actions, GitHub Pages.

**Reference spec:** `docs/superpowers/specs/2026-06-19-docs-site-design.md`

### Mechanical edits applied to every migrated page

For each page, after copying its source lines verbatim:

1. **Drop the source heading line** (the `### Foo` or `## Foo` that started the block). mdBook renders the page title from `SUMMARY.md`; do not repeat it as an in-body heading.
2. **Promote remaining headings by one level** so the deepest structure still nests correctly under a single page (e.g. a former `#### Sub` becomes `### Sub`). Most pages have no sub-headings; only promote if they exist. Do NOT add a new H1 — let SUMMARY supply the title.
3. **Rewrite cross-links** using the anchor map below.
4. Leave all other prose, tables, and code blocks **verbatim**.

## Task 11: Add the GitHub Pages deploy workflow

**Files:**
- Create: `.github/workflows/docs.yml`

- [ ] **Step 1: Create the workflow**

```yaml
name: Docs

on:
  push:
    branches: [main]
    paths:
      - "docs/book/**"
      - "README.md"
      - ".github/workflows/docs.yml"
  workflow_dispatch:

# Allow the workflow to publish to GitHub Pages.
permissions:
  contents: read
  pages: write
  id-token: write

# Only one concurrent Pages deployment; don't cancel an in-progress deploy.
concurrency:
  group: pages
  cancel-in-progress: false

env:
  MDBOOK_VERSION: "0.4.40"

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - name: Install mdBook
        run: |
          mkdir -p "$HOME/.local/bin"
          curl -sSL "https://github.com/rust-lang/mdBook/releases/download/v${MDBOOK_VERSION}/mdbook-v${MDBOOK_VERSION}-x86_64-unknown-linux-gnu.tar.gz" \
            | tar -xz -C "$HOME/.local/bin"
          echo "$HOME/.local/bin" >> "$GITHUB_PATH"
      - name: Build book
        run: mdbook build docs/book
      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: docs/book/book

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

`MDBOOK_VERSION` MUST match the version installed in Task 1 Step 1.

- [ ] **Step 2: Validate the YAML locally**
